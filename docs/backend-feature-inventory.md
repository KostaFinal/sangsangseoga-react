# 상상서가 — 현재 페이지 기준 백엔드 연동 필요 기능

로그인/회원가입, 책 만들기 등은 나중에 통합 예정이라 제외하고, **지금 있는 페이지**에서 실제로 동작하는 기능만 정리했습니다.

---

## 1. 친구의 서재 (메인 책 목록 / 상세)
- 책 목록 조회: 장르 필터(전체/소설/시/에세이/동화/지식정보), 검색(제목/작가/장르/요약), 정렬(최신순/인기순/좋아요순), 페이지네이션
- 책 상세 조회
- 좋아요 토글
- 북마크(내 서재 보관) 토글
- 책 삭제
- 댓글 작성
- 댓글에 답글 작성 (1단계 depth)
- 책 신고 (사유 선택 + 상세 내용)
- 댓글 신고

## 2. 작가 소개
- 작가 디렉토리 조회: 검색(이름), 정렬(팔로워순/활동순)
- 작가 프로필 조회: 사진/소개/팔로워수/작품수
- 작가 팔로우/팔로잉 토글
- 작가 신고
- 작가의 작품 목록 — **지금 하드코딩된 카드 3개**, 실제 작가별 작품 데이터 연동 필요 (별도로 짚었던 부분)

## 3. 책 뷰어
- 5종 뷰어: 소설 / 시 / 에세이 / 동화 / 지식정보 (장르별 다른 렌더링, 데이터는 공통 `book.pages` 기반)
- 페이지별 독서 메모 작성/수정/삭제 (책ID + 페이지 인덱스 단위)
- 완독 처리 → 완독 화면 진입
- 완독 후 댓글(리뷰) 작성 — 1번의 댓글 작성과 동일 로직, 경로만 다름 (통합 고려)
- 완독 후 추천 작품 — 지금 하드코딩 3개, 실데이터 연동 필요

---

## 데이터 모델 메모
- **Book**: id, title, author, genre, summary, coverImage, likes, isLikedByMe, isBookmarked, commentsCount, comments[], pages[], createdAt
- **Comment**: id, user, text, date, replies[]
- **Author**: name, avatar, followers, worksCount, bio (지금 일부는 하드코딩 registry, 일부는 책 author 필드에서 역산)
- **Report**: targetType(comment/book/author), targetId, reason, detail, date
- **Memo**: bookId, pageKey, text

## 우선순위 제안
1. 책 CRUD + 좋아요/북마크/삭제
2. 작가 팔로우 영속화 + 작가 작품 실데이터 연동 (지금 가장 눈에 띄는 가짜 데이터)
3. 댓글/답글
4. 신고 처리(운영자 검토 큐)
5. 메모 / 완독 상태 영속화
